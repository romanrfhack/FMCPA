# Security Track

## Objetivo
- Definir y preparar el endurecimiento transversal de acceso a la plataforma despues del cierre funcional del MVP.

## Alcance propuesto
- Autenticacion
- Base de autorizacion futura por roles o perfiles
- Proteccion de acceso a archivos y evidencias
- Criterios minimos de seguridad operativa para la app web

## Fuera de alcance
- Integraciones externas de identidad no justificadas todavia
- Seguridad avanzada de infraestructura no documentada en el repo
- Cambios de CI/CD fuera de una necesidad claramente aprobada

## Punto de partida real
- El MVP no incluye autenticacion completa.
- Tampoco incluye autorizacion completa ni un modelo de roles cerrado.

## Entregables esperados
- Propuesta de autenticacion inicial
- Base conceptual de autorizacion futura por roles
- Lineamientos para acceso seguro a evidencias y adjuntos
- Lista ordenada de endurecimientos tecnicos posteriores al MVP

## Criterios de salida sugeridos
- Existe una definicion clara del siguiente paso de seguridad.
- Queda separado lo imprescindible para operar de lo que puede esperar a una fase posterior.

## Dependencias
