# Working Agreement

## Objetivo
Definir como se trabajara este repositorio en futuras sesiones con Codex.

## Acuerdos vigentes
- El trabajo se realizara por etapas aprobadas.
- Antes de implementar cambios funcionales debe existir aprobacion explicita de la etapa correspondiente.
- Cada sesion debe actualizar, cuando aplique, `current-phase.md`, `session-log.md`, `decisions.md`, `risks.md`, `backlog.md` y `prompts-log.md`.
- No se debe inventar funcionalidad fuera del contexto documentado y aprobado.
- La documentacion de control se mantendra en espanol.
- Las decisiones relevantes deben registrarse antes o junto con el cambio de direccion.
- Los riesgos y pendientes deben quedar trazables al cierre de cada sesion.

## Limites operativos actuales
- No modificar codigo funcional de Angular, .NET ni base de datos dentro de esta linea base.
- No modificar configuracion de build, CI/CD ni dependencias dentro de esta linea base.
- Mantener la organizacion documental bajo `/docs`.

## Flujo esperado por sesion
1. Revisar fase actual, backlog, decisiones y riesgos.
2. Ejecutar solo trabajo alineado con la etapa aprobada.
3. Registrar avances y acuerdos de la sesion.
4. Dejar un cierre claro para la siguiente sesion.
